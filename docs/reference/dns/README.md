# Документация по протоколу DNS

Добро пожаловать в документацию по протоколу DNS (Domain Name System). Эта документация предназначена для новичков, которые хотят понять основы работы DNS.

## Что такое DNS?

DNS (Domain Name System) — это система, которая преобразует человекочитаемые имена доменов (например, `google.com`) в IP-адреса (например, `142.250.185.14`), которые компьютеры используют для связи друг с другом.

Представьте DNS как телефонную книгу интернета: когда вы хотите позвонить кому-то, вы ищете имя в телефонной книге, чтобы найти номер телефона. Точно так же, когда вы вводите адрес веб-сайта в браузере, DNS находит соответствующий IP-адрес.

## Зачем нужен DNS?

Без DNS нам пришлось бы запоминать числовые IP-адреса для каждого веб-сайта. Вместо того чтобы вводить `142.250.185.14`, мы можем просто ввести `google.com` — намного проще и понятнее!

## Основные RFC документы

DNS протокол описан в следующих RFC (Request for Comments) документах:

- **RFC 1034** — Domain Names — Concepts and Facilities (основные концепции)
- **RFC 1035** — Domain Names — Implementation and Specification (техническая спецификация)
- **RFC 3596** — DNS Extensions to Support IP Version 6 (поддержка IPv6)
- **RFC 6891** — Extension Mechanisms for DNS (EDNS0)
- **RFC 8484** — DNS Queries over HTTPS (DoH)
- **RFC 7858** — Specification for DNS over Transport Layer Security (DoT)
