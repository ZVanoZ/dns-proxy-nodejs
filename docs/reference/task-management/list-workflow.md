# Workflow со списком задач: Работа с предварительно составленным списком

**Дата создания:** 2026-01-22  
**Версия методологии:** vanzan01/cursor-memory-bank v0.8

---

## Описание подхода

Этот подход предполагает, что пользователь **заранее составляет полный список задач** в файле `todo-list.md`, а затем **берет их в работу по одной**, в любом порядке. Каждая задача из списка обрабатывается независимо через полный цикл методологии.

---

## Ключевые принципы

1. ✅ **Можно заранее составить полный список задач** в файле `todo-list.md`
2. ✅ **Задачи из списка можно брать в работу по одной**, в любом порядке
3. ✅ **Не обязательно обрабатывать все задачи сразу**
4. ✅ **Каждая задача из списка обрабатывается независимо** через полный цикл `/van → /plan → [/creative] → /build → /reflect → /archive`
5. ✅ **После завершения одной задачи можно перейти к следующей** из списка

---

## Рекомендуемый подход

### Шаг 1: Составление списка задач

Создайте или обновите файл `todo-list.md` со списком всех задач:

```markdown
Список задач, которые нужно добавить в план и контекст для них.

1. Используя методологию vanzan01/cursor-memory-bank v0.8 составить список задач.
   См. [geminy.md](001/geminy.md)

2. Улучшить и актуализировать документацию
   см. [question.md](002/geminy/question.md) и [answer.md](002/geminy/answer.md)

3. Актуализировать код и стек. см. [geminy.md](001/geminy.md)

4. Добавить поддержку протокола IPv6

5. Добавить сбор и журналирование статистики
```

**Рекомендации:**
- Нумеруйте задачи для удобства
- Добавляйте ссылки на контекстные файлы, если они есть
- Описывайте задачи достаточно подробно для понимания

---

### Шаг 2: Выбор задачи из списка

Выберите задачу из `todo-list.md`, которую хотите взять в работу. Можно выбирать в любом порядке, не обязательно по порядку номеров.

---

### Шаг 3: Инициализация задачи через команду `/van`

Используйте команду `/van` с описанием выбранной задачи:

```
/van [описание задачи из списка]

Пример:
/van Улучшить и актуализировать документацию проекта dns-proxy-nodejs. Цель: Очистить README.md, актуализировать инструкции по запуску и структурировать папку /docs. Контекст задачи в файлах question.md и answer.md.
```

**Важно:**
- Включите достаточно контекста в описание задачи
- Если есть ссылки на контекстные файлы, упомяните их в описании
- AI прочитает задачу и определит уровень сложности

---

### Шаг 4: Выполнение полного цикла методологии

После инициализации задачи через `/van`, выполните полный цикл методологии:

```
/plan → [/creative] → /build → /reflect → /archive
```

Этот цикл идентичен стандартному workflow (см. [standard-workflow.md](standard-workflow.md)).

---

### Шаг 5: Переход к следующей задаче

После выполнения `/archive` для текущей задачи:

1. Вернитесь к `todo-list.md`
2. Выберите следующую задачу из списка
3. Повторите шаги 3-4 для новой задачи

---

## Пример полного цикла работы со списком задач

### Исходный список в `todo-list.md`:

```markdown
1. Добавить поддержку протокола IPv6
2. Улучшить и актуализировать документацию
3. Актуализировать код и стек
4. Добавить сбор и журналирование статистики
```

### Работа с задачей #2 (выбрана первой):

```
/van Улучшить и актуализировать документацию проекта dns-proxy-nodejs. Цель: Очистить README.md, актуализировать инструкции по запуску и структурировать папку /docs.

/plan

/build

/reflect

/archive
```

### Работа с задачей #1 (выбрана второй):

```
/van Добавить поддержку протокола IPv6 в DNS-прокси сервер. Необходимо обрабатывать AAAA записи и поддерживать IPv6 адреса в конфигурации.

/plan

/creative  # Вероятно потребуется

/build

/reflect

/archive
```

И так далее для остальных задач.

## Преимущества подхода со списком задач

1. **Планирование:** Можно заранее спланировать все задачи проекта
2. **Гибкость:** Можно выбирать задачи в любом порядке
3. **Контекст:** Все задачи и их контекст хранятся в одном месте
4. **Независимость:** Каждая задача обрабатывается независимо
5. **Отслеживание:** Легко видеть, какие задачи выполнены, а какие остались

---

## Отличия от стандартного подхода

| Аспект | Стандартный подход | Подход со списком задач |
|--------|-------------------|------------------------|
| **Источник задачи** | Создается на лету через `/van` | Берется из `todo-list.md` |
| **Планирование** | Задача планируется сразу | Список задач планируется заранее |
| **Порядок выполнения** | Задачи выполняются по мере появления | Задачи можно выбирать в любом порядке |
| **Контекст** | Контекст добавляется при создании задачи | Контекст хранится в `todo-list.md` и связанных файлах |

---

## Рекомендации

1. **Обновляйте список:** После завершения задачи можно отметить её в `todo-list.md` (например, добавить ✅)
2. **Добавляйте контекст:** Используйте ссылки на контекстные файлы для лучшего понимания задач
3. **Группируйте задачи:** Можно группировать связанные задачи в списке
4. **Приоритизируйте:** Можно упорядочить задачи по приоритету в списке

---

## Важные замечания

- ⚠️ **Одна активная задача:** В `memory-bank/tasks.md` всегда только одна активная задача
- ⚠️ **Независимость:** Каждая задача обрабатывается независимо через полный цикл
- ⚠️ **Архивирование:** После `/archive` задача завершается, и можно перейти к следующей
- ⚠️ **Контекст:** Контекстные файлы должны быть доступны при работе с задачей
