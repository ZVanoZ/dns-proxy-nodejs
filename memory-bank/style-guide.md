# Memory Bank: Style Guide

## Стиль кода

### TypeScript
- **Строгая типизация (ОБЯЗАТЕЛЬНО)** - использование strict mode во всех `*.ts` файлах проекта
  - Все переменные, параметры функций и возвращаемые значения должны иметь явные типы
  - Запрещено использование `any` без явной необходимости и обоснования
  - Использование `unknown` вместо `any` при необходимости работы с неизвестными типами
  - Строгая типизация применяется ко всем файлам проекта, включая тесты
- **ESM модули** - import/export синтаксис
- **Объектно-Ориентированное Программирование (ООП)** - основной подход для всего кода приложения
  - Использование классов для организации логики
  - Инкапсуляция через модификаторы доступа (protected, private, public)
  - Паттерны: Factory, Service, Singleton
  - **Исключение**: Unit-тесты используют функциональный подход через Jest
- **Асинхронность** - async/await для асинхронных операций

### Именование
- **Классы**: PascalCase (App, LoggerFactory)
- **Функции/методы**: camelCase (loadFromIni, setLogger)
- **Файлы**: camelCase для модулей (OptionsLoader.ts)
- **Директории**: PascalCase для основных модулей (App/, Logger/)

### Структура кода
- Разделение на модули по функциональности
- Использование фабрик для создания объектов
- Сервисы для бизнес-логики

### Комментарии
- Комментарии на русском языке (судя по коду)
- Отладочные console.debug для разработки
- Структурированное логирование для продакшена

### Обработка ошибок
- Try-catch для асинхронных операций
- Логирование ошибок
- Выход из процесса при критических ошибках

### Конфигурация
- Использование INI файлов для настроек
- Поддержка различных окружений через переменные окружения
- Шаблоны конфигурации (.dist файлы)
