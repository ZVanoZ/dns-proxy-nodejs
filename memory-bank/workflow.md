# Использование "vanzan01/cursor-memory-bank" в проекте

## Инициализация

### 1. Копирование базовых файлов

```bash
git clone https://github.com/vanzan01/cursor-memory-bank.git ./tmp/cursor-memory-bank
cp -vR ./tmp/cursor-memory-bank/.cursor/* ./.cursor/
```

### 2. Инициализация "vanzan01/cursor-memory-bank"

1. Откройте новый чат AI в режиме "Agent"
2. Введите команду `/van` и нажмите запустить

## Структура Memory Bank

- `tasks.md` - текущая задача и план реализации
- `activeContext.md` - активный контекст работы
- `projectbrief.md` - краткое описание проекта
- `progress.md` - прогресс реализации
- `productContext.md` - контекст продукта
- `systemPatterns.md` - архитектурные паттерны
- `techContext.md` - технический контекст
- `style-guide.md` - руководство по стилю кода
- `archive/` - архив завершенных задач
- `creative/` - творческие решения и дизайн
- `reflection/` - рефлексия по завершенным задачам

## Работа с задачами

Список задач находится в `docs/tasks/index.md`. Для работы с задачами используйте команды:

- `/van <task_id>` — инициализация задачи (анализ сложности, маршрутизация)
- `/plan [task_id]` — планирование задачи (детальный план, утверждение)
- `/creative [task_id]` — творческая фаза (архитектурные и дизайн-решения)
- **VAN QA** — техническая валидация перед реализацией (см. ниже)
- `/build` — реализация по плану
- `/reflect` — рефлексия по задаче
- `/archive` — архивирование задачи

Подробнее см. [Инструкция по добавлению задач в план](docs/reference/task-management/index.md).

### VAN QA (техническая валидация)

**Что это:** режим технической проверки *после* CREATIVE и *перед* BUILD. Убеждает, что зависимости, конфигурация, окружение и минимальная сборка в порядке, прежде чем приступать к реализации.

**Когда использовать:** после завершения творческой фазы (`/creative`), когда план и дизайн-решения уже зафиксированы и нужно проверить готовность к BUILD.

**Как запустить:** в чате напечатать **`VAN QA`** (без слэша) и отправить сообщение. Или упомянуть «запусти VAN QA» в запросе.

**Что проверяется (четыре шага):**
1. **Зависимости** — все нужные пакеты установлены и совместимы.
2. **Конфигурация** — формат и совместимость конфигов (например, app.ini).
3. **Окружение** — наличие инструментов сборки, права доступа.
4. **Минимальная сборка** — сборка и базовый прогон тестов/запуск проходят.

**Результат:**
- Если все проверки пройдены — можно переходить к **BUILD** (`/build`).
- Если есть ошибки — исправить проблемы и снова запустить **VAN QA**.

В проекте правила VAN QA лежат в `.cursor/rules/isolation_rules/visual-maps/van_mode_split/` (van-qa-main.mdc и папки van-qa-checks, van-qa-utils).
